# API 3.1 - 获取公开作品画廊

## 接口信息

- **接口名称**: 获取公开作品画廊
- **请求方法**: GET
- **请求路径**: `/api/works/gallery`
- **需要认证**: 否

---

## 接口描述

获取所有公开发布的作品列表，支持分页、类型筛选和排序。

---

## 请求参数

### Query参数
| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| page | Integer | 否 | 页码 | 1 |
| pageSize | Integer | 否 | 每页数量 | 20 |
| type | String | 否 | 作品类型(story/music/art/poem) | 全部 |
| sortBy | String | 否 | 排序字段(created_at/likes/views) | created_at |
| order | String | 否 | 排序方式(asc/desc) | desc |

### 请求示例

```
GET /api/works/gallery?page=1&pageSize=20&type=story&sortBy=likes&order=desc
```

| views_count | Integer | 浏览数 |
| created_at | DateTime | 创建时间 |

| author.id | UUID | 作者ID |
| author.nickname | String | 作者昵称 |
| author.avatar | String | 作者头像 |

---

## 业务规则

1. 仅返回状态为published的作品
2. 默认按创建时间倒序排列
3. 支持按点赞数、浏览数排序
4. 浏览该接口不增加作品浏览数
5. 作品类型枚举：story, music, art, poem

---

## 使用场景

- 首页作品展示
- 作品广场浏览
- 发现优秀作品
- 分类作品浏览

---

## 注意事项

1. 建议使用缓存提高响应速度
2. 热门作品可以单独缓存
3. 大量数据时注意分页性能
4. 图片URL建议使用CDN
