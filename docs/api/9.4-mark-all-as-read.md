# API 9.4 - 标记所有通知为已读

## 接口信息

- **接口名称**: 标记所有通知为已读
- **请求方法**: PUT
- **请求路径**: `/api/notifications/read-all`
- **需要认证**: 是

---

## 接口描述

一键将当前用户的所有未读通知标记为已读状态。

---

## 请求参数

### Header参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | String | 是 | Bearer {accessToken} |

### 请求示例

```http
PUT /api/notifications/read-all
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

---

## 响应结果

### 成功响应 (200 OK)

```json
{
  "success": true,
  "message": "已标记所有通知为已读",
  "data": {
    "markedCount": 15,
    "timestamp": "2024-12-22T10:30:00.000Z"
  }
}
```

### 错误响应

#### 401 Unauthorized - 未授权
```json
{
  "success": false,
  "message": "请先登录"
}
```

#### 500 Internal Server Error - 服务器错误
```json
{
  "success": false,
  "message": "操作失败"
}
```

---

## 字段说明

### 响应字段
| 字段名 | 类型 | 说明 |
|--------|------|------|
| markedCount | Number | 标记的通知数量 |
| timestamp | DateTime | 操作时间 |

---

## 业务规则

1. 只标记当前用户的未读通知
2. 批量操作，一次性标记所有未读
3. 操作后未读数量归零
4. 具有幂等性，重复调用不会报错
5. 不影响通知的其他状态

---

## 使用场景

- 通知中心"全部已读"按钮
- 清空未读角标
- 批量处理通知
- 快速清理未读消息

---

## 注意事项

1. 操作不可撤销，请谨慎使用
2. 建议提供二次确认提示
3. 操作后应立即刷新未读数量
4. 不会删除通知，仅改变已读状态

---

## Postman测试

### 配置步骤
1. 创建新请求
2. 选择方法：`PUT`
3. 输入URL：`http://localhost:3000/api/notifications/read-all`
4. 在Headers中添加：
   - Key: `Authorization`
   - Value: `Bearer {你的accessToken}`
5. 点击`Send`发送请求

### 预期结果
- 状态码：200
- 返回标记的通知数量
- 未读数量变为0
