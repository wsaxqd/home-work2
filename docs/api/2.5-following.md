# API 2.5 - 获取关注列表

## 接口信息

- **接口名称**: 获取关注列表
- **请求方法**: GET
- **请求路径**: `/api/users/following`
- **需要认证**: 是

---

## 接口描述

获取当前用户关注的用户列表，支持分页查询。

---

## 请求参数

### Header参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | String | 是 | Bearer {accessToken} |

### Query参数
| 参数名 | 类型 | 必填 | 说明 | 默认值 |
|--------|------|------|------|--------|
| page | Integer | 否 | 页码 | 1 |
| pageSize | Integer | 否 | 每页数量 | 20 |

### 请求示例

```
GET /api/users/following?page=1&pageSize=20
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

## 字段说明

### 响应字段
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | UUID | 被关注用户ID |
| nickname | String | 被关注用户昵称 |
| avatar | String | 被关注用户头像URL |
| bio | String | 被关注用户个人简介 |
| followed_at | DateTime | 关注时间 |
| is_follower | Boolean | 对方是否也关注了我 |

---

## 业务规则

1. 按关注时间倒序排列（最近关注的在前）
2. 支持分页查询，默认每页20条
3. is_follower标识是否互相关注
4. 仅返回关注关系未被删除的用户

---

## 使用场景

- 查看我关注的人
- 关注管理页面
- 互相关注状态展示
- 推荐关注功能

---

## 注意事项

1. 关注数量较多时建议使用分页
2. 建议缓存关注数据减少查询
3. is_follower字段可用于展示"互相关注"标识
