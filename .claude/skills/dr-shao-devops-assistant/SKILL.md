## 核心概念

### HTTP vs HTTPS
- **HTTP**: 明文传输，不安全，易被窃听、篡改
- **HTTPS**: 加密传输，需要 SSL 证书，安全可靠
- **端口**: HTTP 使用 80，HTTPS 使用 443

### 代理技术
- **正向代理**: 隐藏访问方身份（如 VPN）
- **反向代理**: 隐藏被访问方身份（如 Nginx）
- **用途**: 负载均衡、安全防护、缓存加速

### 生产环境 vs 开发环境

| 特性 | 开发环境 | 生产环境 |

## 常见问题解决

### 问题 1: 白屏或 502 错误

**排查步骤**：
1. 检查 Nginx 错误日志：`docker logs [nginx容器名]`
2. 检查后端服务状态：`docker ps`
3. 检查端口映射是否正确
4. 清除浏览器缓存或使用无痕模式
5. 检查 nginx.conf 中的 proxy_pass 地址

### 问题 2: SSL 证书不生效

**排查步骤**：
1. 确认证书路径正确
2. 检查 Nginx 配置中的证书挂载
3. 确认 443 端口已开放
4. 检查证书是否过期
5. 重启 Nginx 容器

### 问题 3: Docker 容器无法启动

**排查步骤**：
1. 查看容器日志：`docker logs [容器名]`
2. 检查端口冲突
3. 检查挂载路径是否存在
4. 检查环境变量是否正确
5. 检查镜像是否拉取成功

## 最佳实践

### 1. 安全原则
- ✅ 生产环境必须使用 HTTPS
- ✅ 不要在代码中硬编码密码
- ✅ 使用 .env 文件管理敏感信息
- ✅ .env 文件不要提交到 Git
- ⚠️ AI 不要直接操作已上线的生产服务器（有幻觉风险）

### 2. 部署原则
- ✅ 使用自动化脚本，避免手动操作
- ✅ 实现无感知升级（毫秒级切换）
- ✅ 保留旧版本镜像，便于回滚
- ✅ 限制日志文件大小，保护服务器资源
- ✅ 设置容器自动重启策略

### 3. 配置管理
- ✅ 开发/生产环境配置分离
- ✅ 使用 .env.template 作为配置模板
- ✅ 生产环境配置手动拷贝，不要推送到 Git
- ✅ 使用环境变量传递配置

### 4. Nginx 优化
- ✅ 启用 Gzip 压缩
- ✅ 配置静态资源缓存
- ✅ 设置合理的超时时间
- ✅ 启用访问日志，便于排查问题
- ✅ 配置错误页面

## 关键命令速查

### Docker 相关
```bash
# 启动服务
docker-compose up -d

# 查看日志
docker logs [容器名]
docker logs -f [容器名]  # 实时查看

# 查看运行中的容器
docker ps

# 进入容器
docker exec -it [容器名] bash

# 停止服务
docker-compose down

# 清理未使用的镜像
docker image prune -a
```

### Nginx 相关
```bash
# 测试配置文件
nginx -t

# 重载配置
nginx -s reload

# 查看错误日志
tail -f /var/log/nginx/error.log
```

### SSL 证书相关
```bash
# 申请证书（Let's Encrypt）
certbot certonly --standalone -d yourdomain.com

# 查找证书位置
find / -name "*.pem" 2>/dev/null

# 检查证书有效期
openssl x509 -in cert.pem -noout -dates
```
