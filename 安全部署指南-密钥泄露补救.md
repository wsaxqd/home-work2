## 🔑 密钥管理

### 新密钥生成

**生成方法**（命令行执行）：
```bash
# 生成 JWT_SECRET
openssl rand -hex 64

# 生成 JWT_REFRESH_SECRET
openssl rand -hex 64

# 生成 SESSION_SECRET
openssl rand -hex 64
```

### 密钥存储规则

**严格遵守以下规则：**

1. **不在 Git 中存储密钥**
   - 所有 `.env*` 文件都在 `.gitignore` 中
   - 定期检查是否有意外提交

2. **不在文档中显示完整密钥**
   - 只显示前8个字符和省略号
   - 例如：`JWT_SECRET=d792640a...`

3. **密钥轮换周期**
   - JWT 密钥：90 天轮换一次
   - 数据库密码：90 天轮换一次
   - 第三方 API 密钥：按服务商要求

4. **访问控制**
   - 仅生产环境管理员可访问完整密钥
   - 通过环境变量或密钥管理服务（如 AWS Secrets Manager）存储
   - 不通过邮件或 Slack 传输密钥

## 🛡️ 6 层安全防护框架

### 1️⃣ 密钥层 - 生成与存储
- [x] 使用安全的随机数生成器
- [x] 密钥长度 >= 256 bit（64 hex 字符）
- [x] 不在代码或文档中硬编码
- [x] 使用环境变量加载

### 2️⃣ 传输层 - 网络通信
- [x] HTTPS/TLS 1.2+ 加密
- [x] 所有 API 调用使用 HTTPS
- [x] 禁用 HTTP 重定向到 HTTPS
- [x] 检查 SSL 证书有效期

### 3️⃣ 应用层 - 业务逻辑
- [x] JWT 令牌签名验证
- [x] 令牌过期检查（7 天）
- [x] 刷新令牌处理（30 天）
- [x] 会话管理和销毁

### 4️⃣ 数据库层 - 数据保护
- [x] 数据库密码复杂（16+ 字符）
- [x] 连接需要 SSL/TLS
- [x] 最小权限原则（只有必要权限）
- [x] 定期备份和加密

### 5️⃣ 基础设施层 - 服务器安全
- [x] 防火墙规则（仅开放 22, 80, 443）
- [x] SSH 密钥认证（禁用密码登录）
- [x] 操作系统定期更新
- [x] 日志监控和告警

### 6️⃣ 监控层 - 检测与响应
- [x] Git 提交监控（防止密钥泄露）
- [x] API 请求日志
- [x] 错误和异常追踪
- [x] 安全事件告警

## 🔍 Git 历史清理

### 状态检查
```bash
# 检查是否还有暴露的密钥
git log --all --full-history -- ".env.production" | wc -l
# 预期结果：0（表示已完全删除）

# 检查是否有高熵字符串
git log -p --all | grep -E '[a-f0-9]{64}' | head -20
# 预期结果：无（或只有非密钥的字符串）
```

### 清理执行
```bash
# 执行清理脚本
./clean-git-history.sh  # Linux/Mac
# 或
.\clean-git-history.bat  # Windows

# 验证清理成功
git push origin --force --all
git push origin --force --tags
```
