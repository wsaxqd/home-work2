| （其他文件） | 待确认 | （待扫描） |

### 具体错误示例
```typescript
// skillTree.ts 第 97 行
export const getMyProgress = (subject?: string) => {
  const params = subject ? { subject } : {};  // ❌ Error
  return api.get<UserSkillProgress[]>('/skill-tree/my-progress', { params });
};

错误信息：
Type 'Record<string, string | number | boolean> | undefined' is not assignable
Property 'subject' is incompatible with index signature.
Type 'undefined' is not assignable to type 'string | number | boolean'.
```

### 问题分析
**根本原因**：TypeScript 严格模式下的类型推导问题
- 当 `subject` 为 `undefined` 时，`params` 对象的类型推导有问题
- API 请求参数类型过于严格

**影响范围**：至少 42 个地方有类似问题

### 通过标准
- ❌ 零 TypeScript 错误（当前：42 个错误）

## 🚨 阻断性问题详情

### 问题 #1：skillTree.ts 的类型不兼容

**文件**：`src/services/api/skillTree.ts`
**行号**：97, 127 等多处
**优先级**：🔴 **紧急** - 阻止上线

**修复建议**：
```typescript
// 修复方案 1：使用 Record<string, any>
const params: Record<string, any> = subject ? { subject } : {};

// 修复方案 2：显式类型注解
const params: Record<string, string | undefined> = subject ? { subject } : {};

// 修复方案 3：使用 URLSearchParams
const params = new URLSearchParams();
if (subject) params.append('subject', subject);
```

## 📞 支持与帮助

**TypeScript 错误修复指南**：
- 问题：`Type 'undefined' is not assignable to type 'string | number | boolean'`
- 原因：对象属性可能为 undefined，但类型声明不允许
- 解决：
  1. 使用可选属性：`{ subject?: string }`
  2. 或使用 `Record<string, any>`
  3. 或显式处理 undefined 情况
