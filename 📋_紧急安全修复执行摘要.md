## 🔐 安全建议

### 密钥管理最佳实践
✅ **正确做法**：
- 所有密钥通过环境变量或密钥管理服务加载
- 每个环境（开发、测试、生产）使用不同的密钥
- 定期轮换密钥（90 天）
- 使用强随机数生成器创建密钥
- 在文档中仅显示密钥的前几个字符和省略号

❌ **错误做法**：
- 硬编码密钥到源代码
- 在 Git 中提交 .env 文件
- 在文档或日志中显示完整密钥
- 在邮件或 Slack 中传输密钥
- 多个环境共用同一密钥

### Git 预防措施
```bash
# 1. 确保 .gitignore 正确配置
.env
.env.local
.env.production
.env.development

# 2. 创建 pre-commit 钩子防止密钥提交
# .git/hooks/pre-commit
if git diff --cached | grep -E "SECRET|PASSWORD|KEY"; then
    echo "❌ 检测到敏感信息！"
    exit 1
fi

# 3. 定期审查 Git 历史
git log --all -p | grep -i "password\|secret\|api_key"
```
